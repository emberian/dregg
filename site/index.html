zero-knowledge · object-capabilities · post-quantum

Authorization that proves without revealing.

Distributed object-capability authorization with ZK proofs. Agents prove they are authorized to act across organizational boundaries — without revealing their delegation chain, the identities of intermediate authorities, or what other capabilities they hold.

Key capabilities

🔐

ZK-private presentation

Present authorization as a STARK proof. The verifier learns only that you are authorized — nothing about your delegation chain, other capabilities, or intermediate authorities.

🔗

Capability attenuation

Capabilities can only be restricted, never amplified. Each delegation step produces a strictly smaller capability set. The type system enforces monotonic narrowing.

🌍

Federated BFT consensus

Small federations (3–64 nodes) share trust roots via adaptive BFT consensus with BLS12-381 threshold signatures. No global chain dependency.

✈️

Offline verification

No blockchain liveness required. Verify a presentation with only the STARK proof, the federation root, and the authorization request.

🤖

Built for AI agents

The execution model (Cells, Turns, Call Forests) is designed for agents that span multiple organizations, each with independent security policies.

🧬

Post-quantum ready

External verification is entirely hash-based (STARK proofs, Merkle commitments). Classical signatures are confined within federation trust boundaries.

⚡

Sub-second proofs

BabyBear STARK with FRI produces ~24 KiB proofs in 300–560ms. Verification is sub-millisecond. Real proofs, not simulations.

📊

Computron budgets

Bounded counters (adapted from Stingray) enable parallel multi-silo execution without per-operation consensus while preventing overspend by Byzantine silos.

Architecture


  SDK Layer ─────────── AgentWallet · AgentRuntime · SiloClient
       │
  Network Layer ─────── TCP wire protocol · STARK verification on receive
       │                 QUIC gossip (Plumtree lazy-push) [planned]
       │
  Federation Layer ──── morpheus: Adaptive BFT (Lewis-Pye & Shapiro)
       │                 hints: BLS12-381 threshold signatures
       │                 federation: Ed25519 consensus, revocation trees
       │
  Coordination Layer ── Causal async DAG · Atomic 2PC · Budget (Stingray)
       │
  Execution Layer ───── cell: Isolated accounts with c-lists
       │                 turn: Atomic transactions (call forests)
       │                 coord: Multi-silo coordination
       │
  Proof Layer ──────── BabyBear STARK + FRI (~24 KiB, sub-second)
       │                 Poseidon2 algebraic hash (SNARK-friendly)
       │                 IVC: hash-chain accumulation
       │
  Commitment Layer ──── 4-ary Merkle trees (BLAKE3 fast / Poseidon2 ZK)
       │                 Fold deltas: monotonic state transitions
       │
  Policy Layer ─────── trace: Datalog evaluator + derivation traces
       │                 token: Macaroon (HMAC) + Biscuit (Ed25519+Datalog)
       │                 tokenizer: X25519-ChaCha20Poly1305 seal/unseal
       │
  Storage Layer ─────── store: redb persistence · secrets: OS keychain
                         audit: Usage log, budget enforcement

Core insight: Capability attenuation is incrementally verifiable computation. Each delegation with restrictions is a fold over a committed fact set. The prover demonstrates a valid chain of attenuations; the verifier checks a single STARK proof.

Quick start

1

Add Pyana to your Cargo.toml:

[dependencies]
pyana-sdk = "0.1"
2

Create a wallet and mint a root capability:

use pyana_sdk::prelude::*;

let wallet = AgentWallet::create()?;
let root = wallet.mint_root(&["dns:read", "compute:execute"])?;
3

Attenuate and present with ZK proof:

let restricted = root.attenuate(Attenuation {
    services: vec![("dns".into(), "read".into())],
    max_ttl: Some(Duration::from_secs(3600)),
    ..Default::default()
})?;

let proof = restricted.prove(&federation_root)?;
// proof is ~24 KiB, verifiable offline in <1ms
