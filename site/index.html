The Problem

Authorization is broken for autonomous agents

Pain

Delegation chains leak organizational structure. Every verifier sees who delegated to whom, through which intermediaries, and what other capabilities exist in the chain.

Pain

Offline verification is impossible in existing systems. If your agents cannot reach the authorization server, they cannot prove anything. Liveness dependency kills autonomy.

Pain

Agent autonomy requires trust in the infrastructure. Exit a platform and you lose your history, your credentials, your proof of work. The infrastructure owns your state.

Pyana

Authorization proofs are zero-knowledge STARKs. The verifier learns exactly one bit: authorized or not. No delegation chain, no intermediate identities, no metadata leakage.

Pyana

Verification is fully offline. A STARK proof plus an attested root is complete. No federation liveness required. Post-quantum security for all external interfaces.

Pyana

Agents own their state as sovereign cells. Your state lives on your machine. The federation stores only a 32-byte commitment. Leave at any time -- your state is already yours.

How It Works

From minting to zero-knowledge verification

01 / Mint

Create root capability

An agent mints a root token bound to key material. This is the unforgeable origin of all downstream authority.

02 / Attenuate

Delegate with narrowing

Capabilities can only be restricted, never amplified. Each delegation step produces a strictly smaller capability set via cryptographic fold.

03 / Prove

Generate ZK proof

The agent produces a ~24 KiB proof that the attenuation chain is valid. Choice of backend: STARK, Kimchi, SP1, or Plonky3. Sub-second generation, post-quantum secure (STARK path).

04 / Verify

One bit: authorized

The verifier checks the proof against an attested root. No online lookup needed. ~2ms verification, fully offline.

Constraint DSL

Write once, prove everywhere

The #[pyana_caveat] and #[pyana_effect] macros compile constraint functions into 8 proof backends from a single source. No manual circuit writing required.

source

Constraint DSL

require! statements, set membership checks, mutations, match arms. Plain Rust syntax with constraint semantics.

backend 1

Rust Evaluator

Direct runtime check. 8 microsecond verification in trusted mode.

backend 2

STARK AIR

BabyBear field, FRI commitment, ~24 KiB proofs. Post-quantum.

backend 3

Kimchi (Mina)

Pasta curves, IPA commitment. Mina-compatible proof format.

backend 4

Midnight ZKIR

Compiles to ZKIR v3 for Cardano/Midnight's proof server.

backend 5

SP1 (RISC-V zkVM)

Proves execution in Succinct's zkVM. Groth16 wrapping for EVM.

backend 6

Plonky3

Native Plonky3 Air impl with recursive composition support.

backend 7-8

Datalog + AIR Descriptor

Policy rules for the Datalog evaluator, plus constraint topology descriptors for tooling.

Privacy Stack

Defense in depth, not a single toggle

Stealth Addresses

Ephemeral per-transaction CellIds derived from meta-addresses (spend + view keys). Unlinkable payments without coordination. Monero/EIP-5564 pattern.

Pedersen Commitments

Homomorphic value commitments over Ristretto. Amounts are hidden but arithmetic is verifiable (balance proofs, range proofs) without revealing values.

Dandelion++ Routing

Stem phase (single-hop relay, 90% probability per hop) before fluff (broadcast). Prevents network-level linkability of transactions to IP addresses.

Encrypted Turns

Turn payloads can be encrypted to specific recipients. The federation orders opaque blobs; only participants can read the content.

Faceted Capabilities

E-language restricted object views. Bitmask on capabilities controls which effect types are accessible. Read-only, transfer-only, admin-only facets.

Sealer/Unsealer Pairs

E-style rights amplification. X25519-ChaCha20Poly1305 pairs with forward secrecy (fresh ephemeral per seal). Partition-tolerant capability transfer.

Primitives

Core abstractions

Cells

Isolated objects with capability lists, balance, nonce, program predicates, and private notes. Sovereign (32-byte commitment) or hosted mode. The unit of identity and state.

Turns

Atomic state transitions from call forests. A turn either fully commits or fully rolls back. Multi-party turns compose across cells. Eventual sends enable pipelining.

Capabilities

Unforgeable references between cells. Created by introduction or minting. Only narrowed, never amplified. Faceted views restrict to specific effect types (E-language semantics).

Factories

EROS-style object factories. A factory's descriptor IS the constructor transparency -- anyone can inspect what capabilities, programs, and state it grants to creations.

Intents

Privacy-preserving capability marketplace. Broadcast needs; wallets privately evaluate using local Datalog; fulfillment proves with STARK.

Sealer/Unsealer

X25519-ChaCha20Poly1305 keypairs for E-style rights amplification. Encrypt under a capability token; only the holder of both capability and key can recover plaintext.

Stealth Addresses

Ephemeral per-transaction CellIds for unlinkable payments. Monero/EIP-5564 pattern: meta-address (spend + view keys), one-time derivation, scan-and-detect.

Receipt Chains

Every committed turn produces a TurnReceipt. The chain IS your state proof. IVC compresses to constant size. Portable across federations.

CDT (Causal DAG)

Multi-silo coordination without global consensus. Bounded counters prevent overspend. 2PC for atomic cross-silo operations.

Get Started

Quick start

1

Build the workspace:

git clone https://github.com/emberian/pyana && cd pyana
cargo build
2

Run the end-to-end demo (federation + token + STARK + turn execution):

cargo run -p pyana-demo-agent
3

Or use the SDK in your own code:

use pyana_sdk::{AgentWallet, AgentRuntime};
use pyana_token::Attenuation;

let wallet = AgentWallet::new();
let root = wallet.mint_token(b"secret-key-material-32-bytes!!!!", "my-service");
let child = wallet.attenuate(&root, &Attenuation {
    services: vec![("inventory".into(), "read".into())],
    max_ttl: Some(std::time::Duration::from_secs(3600)),
    ..Default::default()
}).unwrap();

// The verifier learns ONE BIT: authorized or not.
let proof = wallet.present_private(&child, "inventory", "read").unwrap();
4

Follow the hands-on tutorial or use the TypeScript SDK:

// TypeScript (npm install @pyana/sdk pyana-wasm)
import init from "pyana-wasm";
import { PyanaClient } from "@pyana/sdk";

const wasm = await init();
const client = await PyanaClient.init(wasm);
const token = await client.wallet.mint("api-gateway");
const restricted = await client.wallet.attenuate(token.token, {
  service: "api-gateway", actions: "read", expiresSecs: 3600,
});
const result = await client.wallet.verify(restricted.token, { action: "read" });
