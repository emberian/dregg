object-capabilities + zero-knowledge proofs + federated consensus

Your agents prove what they can do — without revealing how they got there.

Pyana is a distributed object-capability runtime. Agents hold unforgeable capabilities, delegate with attenuation, prove authorization with ZK STARKs, discover each other through privacy-preserving intents, and carry their own state as proof chains. The network is a sealed capability marketplace.

The Problem

Authorization is broken for autonomous agents

Pain

Delegation chains leak organizational structure. Every verifier sees who delegated to whom, through which intermediaries, and what other capabilities exist in the chain.

Pain

Offline verification is impossible in existing systems. If your agents cannot reach the authorization server, they cannot prove anything. Liveness dependency kills autonomy.

Pain

Agent autonomy requires trust in the infrastructure. Exit a platform and you lose your history, your credentials, your proof of work. The federation owns your state.

Pyana

Authorization proofs are zero-knowledge STARKs. The verifier learns exactly one bit: authorized or not. No delegation chain, no intermediate identities, no metadata leakage.

Pyana

Verification is fully offline. A STARK proof plus an attested root is complete. No federation liveness required. Post-quantum security for all external interfaces.

Pyana

Agents own their state as receipt chains. Every committed turn produces a verifiable receipt. Leave any federation at any time, carrying your full history as a constant-size proof.

How It Works

From minting to zero-knowledge verification

01 / Mint

Create root capability

An agent mints a root token bound to key material. This is the unforgeable origin of all downstream authority.

02 / Attenuate

Delegate with narrowing

Capabilities can only be restricted, never amplified. Each delegation step produces a strictly smaller capability set via cryptographic fold.

03 / Prove

Generate STARK proof

The agent produces a ~24 KiB STARK proof that the attenuation chain is valid. Sub-second generation, post-quantum secure.

04 / Verify

One bit: authorized

The verifier checks the proof against an attested root. No online lookup needed. Sub-millisecond verification, fully offline.

Core insight: Capability attenuation is incrementally verifiable computation. Each delegation is a fold over a committed fact set. The prover demonstrates a valid chain of attenuations; the verifier checks a single STARK proof.

Primitives

Core abstractions

Cells

Isolated objects with capability lists, balance, nonce, program predicates, and private notes. The unit of identity and state.

Turns

Atomic state transitions from call forests. A turn either fully commits or fully rolls back. Multi-party turns compose across cells.

Capabilities

Unforgeable references between cells. Created by introduction or minting. Only narrowed, never amplified. Attenuation is the delegation primitive.

Intents

Privacy-preserving capability marketplace. Broadcast needs; wallets privately evaluate using local Datalog; fulfillment proves with STARK.

Sealer/Unsealer

X25519-ChaCha20Poly1305 keypairs. Encrypt under a capability token; only the holder of both capability and key can recover plaintext.

Receipt Chains

Every committed turn produces a TurnReceipt. The chain IS your state proof. IVC compresses to constant size. Portable across federations.

CDT (Causal DAG)

Multi-silo coordination without global consensus. Bounded counters prevent overspend. 2PC for atomic cross-silo operations.

Get Started

Quick start

1

Build the workspace:

git clone https://github.com/emberian/pyana && cd pyana
cargo build
2

Run the end-to-end demo (federation + token + STARK + turn execution):

cargo run -p pyana-demo-agent
3

Or use the SDK in your own code:

use pyana_sdk::{AgentWallet, AgentRuntime};
use pyana_token::Attenuation;

let wallet = AgentWallet::new();
let root = wallet.mint_token(b"secret-key-material-32-bytes!!!!", "my-service");
let child = wallet.attenuate(&root, &Attenuation {
    services: vec![("inventory".into(), "read".into())],
    max_ttl: Some(std::time::Duration::from_secs(3600)),
    ..Default::default()
}).unwrap();

// The verifier learns ONE BIT: authorized or not.
let proof = wallet.present_private(&child, "inventory", "read").unwrap();
