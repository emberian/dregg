Operator Guide

Running a pyana node makes you a participant in a federation -- a notary service that provides ordering, nullifier tracking, discovery, and root anchoring. In sovereign mode, your node verifies proofs and updates 32-byte commitments. It does not store or execute agent state.

What Your Node Does (and Does Not Do)

Your Node DoesYour Node Does NOT
Verify STARK proofs of valid state transitionsExecute transitions on behalf of agents (sovereign mode)
Track nullifiers (double-spend prevention)See cell contents, balances, or capability sets
Provide ordering via BFT consensusKnow transfer amounts (Pedersen committed)
Anchor attested roots (periodic checkpoints)Know transfer recipients (stealth addresses)
Relay discovery (intent gossip)Store full agent state (only 32-byte commitments)

For hosted cells (legacy/convenience mode), the node does execute and store full state. But the default for new cells is sovereign: your node is a notary, not a database.

Architecture at a Glance

pyana-node
  |
  +-- HTTP API (port 8420)       -- Serves extension + SDK clients
  |     /api/authorize           -- Token authorization
  |     /api/turn                -- Proof-carrying turn submission
  |     /api/intent              -- Intent broadcast (discovery)
  |     /api/faucet (devnet)     -- Token dispensing
  |     /status                  -- Health check
  |
  +-- WebSocket (port 8420/ws)   -- Real-time state updates
  |
  +-- Federation Sync            -- Gossip with peers
  |     Morpheus DAG-BFT         -- Consensus (ordering)
  |     Ed25519 signatures       -- Message authentication
  |     Attested roots           -- Periodic commitment checkpoints
  |     Nullifier set            -- Double-spend prevention
  |
  +-- MCP Server (stdio)         -- AI agent tool interface
  |
  +-- Bridge (port 8421)         -- Cross-federation relay
  |
  +-- Storage (redb)             -- Commitment tree + nullifiers
        ~/.pyana/                 -- Data directory

Quick Start

# Build from source
cargo build -p pyana-node --release

# Initialize the data directory (generates keypair)
pyana-node init --data-dir ~/.pyana

# Start the node
pyana-node run --port 8420 --data-dir ~/.pyana

See Node Setup for the full configuration reference.
