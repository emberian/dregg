Cross-Federation Bridge Nodes

A bridge node connects to multiple federations' gossip networks and relays messages between them: attested roots, revocations, receipts, and conditional proof completions. This enables cross-federation token operations.

Running a Bridge

pyana-node bridge \
  --data-dir ~/.pyana \
  --federation-peers "local1:8420,local2:8420" \
  --remote-peers "abcdef01...:remote1.other-fed.io:8420" \
  --port 8421

The --remote-peers format is federation_id_hex:host:port, where the federation ID is the 64-character hex identifier of the remote federation.

EVM Bridge (Base Sepolia)

The EVM bridge wraps Pyana STARK proofs in Groth16 via SP1 for on-chain verification (~200K gas). Operators run the bridge relay to facilitate deposits and withdrawals. 

# Run the EVM bridge relay
pyana-node bridge-evm \
  --data-dir ~/.pyana \
  --rpc-url https://sepolia.base.org \
  --bridge-contract 0x... \
  --private-key-file ./keys/bridge-operator.key \
  --port 8422

# The relay watches for on-chain deposit events and submits
# corresponding note commitments to the federation.

Mina Bridge

The Mina bridge uses native Pickles recursion. Pyana STARK proofs are wrapped in Kimchi circuits over Pasta curves, producing Pickles recursive proofs that Mina validators verify natively. No relay operator is required for proof verification -- the proofs are self-contained. 

# Generate a Mina-compatible proof
pyana proof generate --backend kimchi --turn $TURN_HASH
pyana proof wrap-pickles --input $KIMCHI_PROOF --output mina-proof.bin

# Submit to a Mina zkApp (requires o1js tooling)
# The proof is natively verifiable by Mina's consensus

Midnight/Cardano Bridge

The Midnight bridge operates at Level 1.5: optimistic acceptance with a dispute window. Operators run an attestation relay. 

# Run the Midnight attestation relay
pyana-node bridge-midnight \
  --data-dir ~/.pyana \
  --midnight-endpoint wss://midnight-node.example.com \
  --attestation-interval 100  # attest every 100 blocks
  --bond-amount 50000

Bridge Security Model

  • EVM (Level 2): Trust assumption is SP1/Groth16 soundness. No trusted relay.
  • Mina (Level 2): Trust assumption is Pickles/IPA soundness. No trusted relay.
  • Midnight (Level 1.5): Trust assumption is that at least one honest party will challenge invalid submissions within the dispute window.

All bridges use proof translation, not consensus bridging. There is no multisig, no threshold committee, no trusted set of bridge operators that could collude. The math is the bridge.
