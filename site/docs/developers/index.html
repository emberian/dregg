Developer Guide

Build applications on pyana using the Rust SDK (pyana-sdk crate). The SDK provides a unified entry point for agents that need to hold tokens, generate proofs, submit turns, and interact with federations.

Contents

In this section

  • SDK Quickstart -- Add pyana-sdk to your project, mint a token, generate a proof
  • Tutorial: First App -- Build an API key management service from scratch
  • TypeScript SDK -- Use pyana from Node.js or the browser via @pyana/sdk

  • Service Mesh -- Mount services, discovery, DFA-governed namespaces
  • Bridges (Mina/EVM) -- Bridge to Mina (Pickles), EVM (SP1/Groth16), Midnight
  • Storage -- Quotas, inboxes, erasure coding, rent economics
  • Privacy API -- Anonymous auth, private notes, unlinkable predicates
  • MCP Integration -- AI agent tools via Model Context Protocol
  • WASM Playground -- Browser-based experimentation

Core Types

The SDK re-exports these primary types at the crate root:

use pyana_sdk::{
    // Wallet and runtime
    AgentWallet, AgentRuntime, SubAgent,

    // Tokens
    Attenuation, AuthRequest, AuthToken,
    HeldToken, DelegatedToken, SignedTurn,

    // Verification modes
    VerificationMode, AuthorizationPresentation,
    DisclosureSpec, FactDisclosure, FactIndex,

    // Proof types
    BridgePresentationProof, StarkProofVerifier,
    IvcProof, verify_ivc,

    // Privacy
    AnonymousPresentation, UnlinkablePredicateProof,
    NoteSecret, NoteTransferProof, NonRevocationProof,

    // Cells and turns
    CellId, Ledger, Turn, TurnBuilder, TurnReceipt, Effect,

    // Cryptographic primitives
    BabyBear, PublicKey, Signature,
};

Adding the Dependency

# Cargo.toml
[dependencies]
pyana-sdk = { git = "https://github.com/emberian/pyana", branch = "main" }

The SDK requires Rust 1.94+ (edition 2024). The workspace uses workspace-level dependency resolution, so version pinning happens at the workspace root.
