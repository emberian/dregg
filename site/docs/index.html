Pyana Documentation

Pyana is a sovereign capability runtime. Agents own their state, prove authorization with ZK STARKs, and interact peer-to-peer when possible. The federation acts as a notary -- providing ordering, double-spend prevention, and discovery -- not as a state host.

Choose your path based on what you need to accomplish:

for users Using Pyana Install the browser extension, create an agent identity, manage tokens, authorize actions, and discover capabilities in the intent marketplace. Get started → for operators Running a Node Set up a pyana-node, join a federation as a notary/ordering service, run a cross-federation bridge, and deploy devnet clusters. Node setup → for developers Building on Pyana Use the Rust SDK to mint tokens, generate STARK proofs, submit turns, integrate with AI agents via MCP, and build browser extensions. SDK quickstart → for architects System Architecture The sovereign cell model, three interaction modes (peer-to-peer, federation-mediated, hosted), privacy stack, and proof architecture. Deep dive →
