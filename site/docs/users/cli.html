CLI Reference (pyana)

The pyana command-line tool is a full-citizen interface to the runtime. Every operation available through the SDK or browser extension is also available via the CLI. It is the primary tool for operators, developers, and power users.

Installation

# From source (requires Rust 1.79+)
cargo install --path cli/

# Or from a release binary
curl -sSf https://pyana.io/install.sh | sh

# Verify installation
pyana --version

Subcommands Overview

SubcommandPurpose
pyana cellCreate, inspect, migrate, split, merge cells
pyana turnBuild, sign, submit turns; inspect and verify receipts
pyana capMint, attenuate, delegate, revoke, present, verify capabilities
pyana walletCreate/restore identity, manage keys, check balance
pyana federationJoin/leave federation, check status, list peers
pyana namespaceMount services, resolve names, manage registrations
pyana storageCheck quotas, space bank status, manage erasure shards
pyana directoryManage petnames, edge names, proposed names
pyana proofGenerate, verify, export, compose proofs
pyana routeClassify paths, inspect DFA, test ACLs
pyana doctorDiagnose node health, verify integrity

Wallet Management

# Create a new identity (generates 24-word mnemonic)
pyana wallet create --mnemonic
# Output: writes mnemonic to stdout, saves encrypted key to ~/.pyana/keys/

# Restore from mnemonic
pyana wallet restore --mnemonic "word1 word2 ... word24"

# Check balance
pyana wallet balance
# Output: 45000 computrons (available), 5000 (locked in deposits)

# Export public key
pyana wallet pubkey
# Output: ed25519:a1b2c3d4...

Cell Operations

# Create a sovereign cell (federation stores only your 32-byte commitment)
pyana cell create --sovereign --federation devnet.pyana.io:8420

# Create a hosted cell (federation stores full state)
pyana cell create --hosted --federation devnet.pyana.io:8420

# Inspect a cell
pyana cell inspect $CELL_ID
# Output: state slots, c-list entries, sovereignty mode, last turn, balance

# Migrate (teleport) to another federation
pyana cell migrate --cell $CELL_ID --to other-fed.io:8420

# Split a cell into children
pyana cell split --cell $CELL_ID --partitions 3

# Merge children back
pyana cell merge --cells $CHILD1,$CHILD2,$CHILD3

Capability Operations

# Mint a root capability token
pyana cap mint --service my-service --root-key ./keys/root.key

# Attenuate (restrict) a token
pyana cap attenuate --token $TOKEN_ID \
  --restrict service=my-service,action=read \
  --ttl 1h --budget 500

# Delegate to another agent
pyana cap delegate --token $TOKEN_ID --to $BOB_PUBKEY \
  --restrict action=read --ttl 24h

# Revoke a delegation
pyana cap revoke --token $TOKEN_ID --delegation $DELEG_HASH

# Generate a ZK presentation proof
pyana cap present --token $TOKEN_ID --request '{"service":"dns","action":"read"}' \
  --mode private

# Verify a received proof
pyana cap verify --proof ./proof.bin --federation-root $ROOT_HEX

Turn Operations

# Build and submit a turn
pyana turn build --target $CELL_ID --effect set-field:counter=42
pyana turn submit --signed

# Verify receipt chain integrity
pyana turn verify-chain --cell $CELL_ID

# Inspect a specific receipt
pyana turn receipt --hash $TURN_HASH
# Output: pre_state, post_state, effects, computron_cost, timestamp

# Export IVC-compressed state proof
pyana turn export-ivc --cell $CELL_ID --output ./state-proof.bin

Federation Operations

# Check federation status
pyana federation status
# Output: connected peers, current epoch, attested root, block height

# Join a federation
pyana federation join --url devnet.pyana.io:8420 --stake 100000

# Leave a federation (keeps your IVC proof)
pyana federation leave

# List peers
pyana federation peers
# Output: node_id, address, last_seen, blocks_produced

Namespace and Service Mesh

# Mount a service at a namespace path
pyana namespace mount --path /services/inference --descriptor ./service.json

# Resolve a service path
pyana namespace resolve /services/inference
# Output: cell_id, descriptor, ACL policy

# List services under a prefix
pyana namespace list /services/

# Register a petname (local alias)
pyana directory petname set bob ed25519:a1b2c3d4...
pyana directory petname get bob
# Output: ed25519:a1b2c3d4...

# Manage name rental
pyana namespace rent --path /public/myname --epochs 12
pyana namespace status --path /public/myname

Storage Operations

# Check storage quota and usage
pyana storage status
# Output: used 45 KiB / 10 MiB quota, rent: 45 computrons/epoch

# Check space bank allocation
pyana storage bank
# Output: bank_id, total allocation, used, available

# Enable erasure coding for a cell
pyana storage erasure enable --cell $CELL_ID --shards 7 --threshold 4

# Check erasure shard health
pyana storage erasure status --cell $CELL_ID

# Inspect inbox (MerkleQueue)
pyana storage inbox
# Output: 3 pending messages, total deposit held: 1500 computrons

Proof Operations

# Generate a proof using a specific backend
pyana proof generate --backend plonky3 --turn $TURN_HASH
pyana proof generate --backend kimchi --turn $TURN_HASH

# Compose multiple proofs
pyana proof compose --mode chain --inputs proof1.bin,proof2.bin,proof3.bin

# Wrap for Mina (STARK-in-Pickles)
pyana proof wrap-pickles --input $STARK_PROOF --output mina-proof.bin

# Wrap for EVM (SP1 Groth16)
pyana proof wrap-sp1 --input $STARK_PROOF --output evm-proof.bin

# Benchmark a proof backend
pyana proof bench --backend plonky3 --circuit effect-vm --turns 100

Route Classification

# Test a path against the DFA classifier
pyana route classify /services/inference/gpt4
# Output: ACCEPT (policy: compute-tier-1, ACL: [cell_id_1, cell_id_2])

# Inspect the DFA state machine
pyana route inspect --states
# Output: 47 states, 312 transitions, 23 accept states

# Propose a constitutional amendment (new route)
pyana route propose --add /services/new-category --acl open
# Output: proposal_id (requires h-rule acceptance)

Diagnostics

# Full health check
pyana doctor --full
# Checks: node connectivity, chain integrity, peer count,
#          disk usage, proof generation, key accessibility

# Quick connectivity test
pyana doctor --connectivity

# Verify your receipt chain from genesis
pyana doctor --verify-chain

Configuration

The CLI reads configuration from ~/.pyana/config.toml: 

[federation]
default = "devnet.pyana.io:8420"

[wallet]
keyfile = "~/.pyana/keys/default.key"

[proof]
default_backend = "plonky3"

[storage]
data_dir = "~/.pyana/data"

All settings can be overridden with environment variables (PYANA_FEDERATION_DEFAULT, etc.) or command-line flags.
