Quickstart

Get up and running with pyana in under 5 minutes. By the end you will have an agent identity, a capability token, and will have authorized an action with a zero-knowledge proof.

Prerequisites

  • A Chromium-based browser (Chrome, Edge, Brave, Arc) or Firefox
  • A running pyana-node on localhost (see Node Setup), or access to a public devnet

Step 1: Install the Extension

The pyana wallet lives as a browser extension. It manages your keys, tokens, and proof generation without exposing secrets to web pages.

  1. Download the latest .zip from the releases page.
  2. Navigate to chrome://extensions (or equivalent for your browser).
  3. Enable "Developer mode" and click "Load unpacked".
  4. Select the unzipped extension/ directory.

You should see the pyana icon in your browser toolbar.

Step 2: Create Your Agent Identity

Click the pyana extension icon to open the wallet popup. On first launch, it will prompt you to create or restore an identity.

  1. Click "Create New Identity".
  2. The wallet generates a 24-word mnemonic. Write it down securely -- this is your backup. The mnemonic derives your Ed25519 signing key via BLAKE3-based HD key derivation at path pyana/0.
  3. Confirm the mnemonic and set an unlock passphrase.

Your public key (agent ID) is now displayed in the wallet. This is your identity across all federations.

Step 3: Connect to a Federation

Open the wallet settings and enter the federation node URL. If you are running a local node, this is http://127.0.0.1:8420 by default.

The extension connects to the node's HTTP API and WebSocket endpoint for real-time state updates. The status indicator should turn green once connected.

Step 4: Receive a Capability Token

Tokens are delegated to you by other agents. For testing, the devnet faucet can issue you a root token: 

# From another terminal (or ask the devnet operator):
curl -X POST http://127.0.0.1:8420/api/faucet \
  -H "Content-Type: application/json" \
  -d '{"recipient": "YOUR_PUBLIC_KEY_HEX", "service": "demo"}'

The token appears in your wallet's "Tokens" tab. It shows the service name, any restrictions (caveats), and whether it is a root token or attenuated.

Step 5: Authorize an Action

When a web page calls window.pyana.authorize(), the extension prompts you to approve. The page specifies what service and action it needs: 

// In the web page:
const result = await window.pyana.authorize({
  service: "demo",
  action: "read",
  mode: "private"  // Zero-knowledge: verifier learns only yes/no
});

// result.proof contains the STARK proof (~24 KiB)
// result.conclusion is true/false

The wallet:

  1. Shows you which token will be used and what the page is requesting.
  2. You approve (or deny).
  3. The wallet generates the STARK proof locally (sub-second).
  4. The proof is returned to the page. Your token contents, delegation chain, and identity are NOT revealed to the verifier.

The privacy mode ("trusted", "selective", or "private") determines how much the verifier learns. See Privacy Modes for details.
