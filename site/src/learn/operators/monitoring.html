Monitoring

pyana-node uses the tracing crate for structured logging. Control log levels via the RUST_LOG environment variable.

Status Endpoint

curl http://127.0.0.1:8420/status

Returns node health, peer count, latest attested root, and sync state.

Log Configuration

# Default: info-level pyana logs
RUST_LOG=pyana_node=info pyana-node run ...

# Debug everything
RUST_LOG=debug pyana-node run ...

# Specific crate tracing
RUST_LOG=pyana_node=debug,pyana_federation=info pyana-node run ...

Distributed Tracing

pyana-node emits tracing spans on every turn::execute, federation peer exchange, and STARK verification, but does not currently export them to a collector. The tracing-subscriber layer stack is local: stdout / JSON logs only. Bridging to OTLP / OpenTelemetry is on the roadmap once a deployment recipe stabilises around a specific collector — until then, ingest stdout JSON with the log shipper of your choice.
