dregg is a big system, but you don't need to understand all of it to be productive. Pick the path that matches what you're trying to do.

I want to use it Users Install the CLI, set up your cipherclerk, connect to a node, and start sending turns. Five minutes to productive. Quickstart → I want to build on it Developers Use the SDK, build an AppServer, register in the nameservice, accept CapTP connections, and deploy programmable queues. Build apps → I want to run it Operators Run a federation node, operate a relay, monitor health, and manage genesis configuration. Node setup → I want to understand it Architecture Deep dive into the blocklace, Effect VM, CapTP, proof system, privacy stack, and trust model. Deep dive →
