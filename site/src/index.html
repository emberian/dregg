What is Pyana?

A few ideas, taken seriously

Sovereignty spectrum

Run a full node on your laptop. Delegate offline-capable snapshots to your phone. Join a reference group for ordering and discovery. Switch modes without ceremony. Your state is yours — always verifiable, always portable.

Proof-carrying state

Every turn is proven by a 24-effect VM in a single STARK. BabyBear field + FRI, sub-second generation, post-quantum secure. You don't trust a validator — you verify a proof. Offline, on-chain, or anywhere in between.

Capabilities everywhere

CapTP carries unforgeable references across the fabric. Sturdy refs survive reboots. Three-party handoff lets Alice introduce Bob to Carol without a directory. Distributed GC cleans up when references drop. It's object-capability networking, end to end.

Privacy by design

Three verification modes from the same Datalog rules: trusted, selective disclosure, or fully private. Stealth addresses, Pedersen commitments, ring membership proofs. You choose what to reveal. The verifier learns only what you decide.

How it works

Cells, turns, proofs

01 / Cells

Isolated objects with capability lists

Each cell holds a c-list, balance, nonce, and optional programs. Cells are confined: they can only reference what is in their c-list.

02 / Turns

Atomic state transitions

A call forest executes depth-first. Multi-party turns compose across cells. If any action fails, everything rolls back.

03 / Effect VM

24 effects, one STARK per turn

Transfer, seal, factory creation, CapTP handoff, queue operations — all proven in a single trace. Sub-second generation.

04 / Fabric

Emergent reference groups

The blocklace is one DAG. Groups form from mutual acknowledgment. Finality emerges from supermajority within a group.

Get Started

Five minutes to a running node

1

Clone and build:

git clone https://github.com/emberian/pyana && cd pyana
cargo build
2

Run a node:

cargo run -p pyana-node run
3

Try the CLI:

pyana cell list
pyana cell create --name my-agent
pyana cap grant <cell-id> --service storage --actions read
pyana namespace register alice --target <cell-id>
4

Run the full demo pipeline:

cargo run -p pyana-demo-agent
