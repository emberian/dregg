object-capabilities + zero-knowledge proofs + blocklace consensus

Your agents prove what they can do — without revealing how they got there.

Pyana is a sovereign capability runtime. Agents hold unforgeable references, delegate with attenuation, prove turns with STARKs, and carry their own state as receipt chains. The federation is a notary, not a host: it orders, anchors roots, detects equivocation. Your cells stay on your machine. The fabric is one DAG.

The Problem

Authorization is broken for autonomous agents

Pain

Delegation chains leak organizational structure. Every verifier sees who delegated to whom, through which intermediaries, and what other capabilities exist in the chain. Your org chart is your authorization model.

Pain

Offline verification is impossible in most systems. If your agents cannot reach the authorization server, they cannot prove anything. Liveness dependency kills autonomy and makes every interaction a privacy leak.

Pain

Exit a platform and you lose your history, your credentials, your proof of work. The federation owns your state. Migration is not a checkbox — it’s a rebuild.

Pyana

Authorization presentations are STARKs over Datalog derivation traces. The verifier can learn exactly what you choose to reveal — from full disclosure down to one bit. No delegation chain, no intermediate identities, no metadata leakage.

Pyana

Verification is fully offline once you hold an attested root. A STARK proof plus a receipt chain is complete. Hash-based crypto end to end; no liveness dependency on any federation between presentation and check.

Pyana

Agents own their cells. Every committed turn produces a TurnReceipt. The chain is your state proof; IVC compresses it to constant size. Leave any federation at any time, carrying your full history.

How It Works

Cells, turns, proofs

01 / Cells

Isolated objects with c-lists

Each cell holds a capability list, a balance, a nonce, optional program predicates, and private notes. Cells are confined: they can only reference what is in their c-list. Sovereign cells live on your machine.

02 / Turns

Atomic state transitions

A call forest executes depth-first. Multi-party turns compose across cells. Either everything commits or everything rolls back. Conservation is enforced; authority must be in-list.

03 / Effect VM

One STARK per turn

Transfer, seal, factory creation, CapTP introduction, queue operations, bridge phases — proven in a single BabyBear+FRI trace. Sub-second generation on a laptop.

04 / Fabric

Blocklace + Cordial Miners

The federation runs one DAG. Cordial Miners closes rounds in 3 messages. Equivocation is structurally detectable on the DAG itself — no accusation protocol, no voting.

Trusted Mode

Full cleartext token + Datalog trace. Same trust domain, internal calls.

~8 us verification

Selective Disclosure

Per-fact reveal / predicate / committed-threshold / hidden, bound by STARK.

~2 ms verification

Fully Private

One bit (allow/deny) + STARK. Optional unlinkable blinding per presentation.

~2 ms verification

Core insight: capability attenuation is incrementally verifiable computation. Each delegation is a fold over a committed fact set. The prover demonstrates a valid chain of attenuations; the verifier checks a single STARK proof against an attested root. The same shape covers turn execution, intent fulfillment, and ring settlement.

Use Cases

Where pyana fits

agents

AI Agent Fleets

Orchestrate thousands of agents across organizational boundaries. Each agent carries proof of its authority. No central authorization server bottleneck, no shared secrets in transit.

privacy

Private Capability Exchange

Broadcast intents ("I need read access to X") without revealing your identity. The trustless solver finds rings; fulfillment is a STARK that leaks nothing about the satisfier’s other capabilities.

Cross-Org Collaboration

Multiple organizations share authorization proofs without exposing internal delegation structures. Selective disclosure lets each party control exactly what they reveal — per-fact.

Primitives

Core abstractions

Cells

Isolated objects with capability lists, balance, nonce, program predicates, and private notes. Sovereign (local + STARK) or hosted (federation-executed). The unit of identity and state.

Turns

Atomic state transitions from call forests. A turn either fully commits or fully rolls back. Multi-party turns compose across cells. Receipts chain via previous_receipt_hash.

Capabilities

Unforgeable references between cells. Created by introduction or minting. Only narrowed, never amplified. Attenuation is the delegation primitive; macaroons and biscuits are wire encodings.

CapTP

Sturdy refs (pyana:// URIs with swiss numbers), sessions, strands, three-party handoff with signed certificates, distributed GC, promise pipelining, store-and-forward over X25519.

Intents

Heterogeneous offers and wants — assets, capabilities, services, storage, namespace entries. Trustless solver finds rings; settlement is escrow-commit-reveal with STARK fulfillment proofs.

Sealer / Unsealer

X25519 + ChaCha20-Poly1305 keypairs. Encrypt under a capability token; only the holder of the matching unsealer can recover plaintext. E-rights pattern, end to end.

Receipt Chains

Every committed turn produces a TurnReceipt with BLAKE3 chain links. Federation receipts add BLS12-381 ThresholdQC aggregation. The chain IS your state proof — portable across federations.

Landscape

What pyana adds

System Pyana’s contribution
UCAN / Biscuit UCAN delegation chains are transparent. Pyana proves the same authorization as a STARK over the Datalog derivation — the verifier learns only the facts you reveal. Biscuit is supported as a wire format.
Cap’n Proto / OCapN Cap’n Proto is serialization + RPC. Pyana adds ocap semantics end to end: cells, turns, three-party handoff, distributed GC, and STARK-verifiable execution that binds the protocol to provable state transitions.
Mina Protocol Shared DNA — zkApp model, call forests, recursive proofs. Pyana applies this shape to authorization and distributed object semantics, not just financial transactions. Pickles is one of our recursion paths.
Midnight / Cardano Privacy DeFi on Cardano. Pyana is a general-purpose ocap runtime where “value” is one kind of capability. We bridge to Midnight via ZKIR v3 (designed) and attestation (implemented).
Cosmos IBC IBC requires active relayer liveness. Pyana verification is fully offline once an attested root is held; the capability model replaces channel-based permissions and the bridge has explicit Lock/Mint/Finalize/Cancel phases.

Get Started

Five minutes to a running node

1

Clone and build the workspace:

git clone https://github.com/emberian/pyana && cd pyana
cargo build
2

Run a single-node federation:

cargo run -p pyana-node run
3

Drive it from the CLI:

pyana cell create --name my-agent
pyana cap grant <cell-id> --service storage --actions read
pyana namespace register alice --target <cell-id>
4

Or use the SDK directly:

use pyana_sdk::{AgentWallet, Attenuation, VerificationMode, AuthRequest};

let mut wallet = AgentWallet::new();
let root = wallet.mint_token(b"secret-key-material-32-bytes!!!!", "inventory");
let child = wallet.attenuate(&root, &Attenuation {
    services: vec![("inventory".into(), "read".into())],
    max_ttl: Some(std::time::Duration::from_secs(3600)),
    ..Default::default()
}).unwrap();

// The verifier learns ONE BIT: authorized or not.
let req = AuthRequest { service: Some("inventory".into()),
                        action: Some("read".into()), ..Default::default() };
let proof = wallet.authorize(&child, &req, VerificationMode::FullyPrivate).unwrap();
