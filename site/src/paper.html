Abstract

What this paper is about

Pyana combines object-capability semantics with zero-knowledge proofs to let agents prove authorization across organizational boundaries without revealing their delegation chain. The key insight is that capability attenuation — monotonically narrowing permissions through delegation — is structurally identical to incrementally verifiable computation. Zero-knowledge presentation emerges naturally from the capability model, not bolted on afterward.

The system achieves sub-second proof generation (~24 KiB STARK proofs in 300–560ms), sub-millisecond verification, offline operation without blockchain liveness, and post-quantum security for external interfaces through exclusively hash-based verification. Agents carry their own state as receipt chains that compress to constant size via IVC, enabling censorship-resistant exit from any federation.

References

Key influences

  • Shapiro & Sela. Cordial Miners: Fast and Efficient Consensus for Every Eventuality. The foundation for pyana's Blocklace federation layer — DAG-based BFT with 3-round finality.
  • Stingray (arXiv:2501.06531). Bounded counters for budget enforcement without per-operation consensus. Adapted for multi-silo coordination.
  • Mina Protocol. zkApp execution model with call forests and cells-as-accounts. Pyana adapts this architecture for authorization and distributed object semantics.
  • Birgisson et al. (Macaroons). HMAC-chained bearer tokens with cryptographic attenuation. Pyana supports both Macaroon and Biscuit formats.
  • Biscuit (Datalog authorization). Ed25519-signed tokens with embedded Datalog policies. Extended with STARK proofs over derivation traces.
  • Mark Miller (object-capabilities). The E programming language and the principle of least authority. Pyana's execution model descends directly from this lineage.
