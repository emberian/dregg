The part nobody else has

It isn’t a library.
It’s a theorem.

The agent runtime is an executable kernel in Lean 4, and its safety properties are proved — not tested. The primitives the demo runs are the ones the proofs are about.

🟢 Non-amplification

An agent can only ever delegate authority it already holds. A delegation that would widen authority is rejected. granted ⊆ held · derive_no_amplify

🟢 Conservation, per-asset

No agent conjures value from nothing — for every asset, even across the escrow holding-store. execFullForestA_conserves_per_asset

🟢 Fail-closed confinement

Out-of-scope is impossible, not merely audited: the executor returns none and the all-or-nothing turn rolls back. recKExecAsset_unauthorized_fails

🟢 Caveats that read state — no TOCTOU

An action can depend on a peer’s state, checked atomically inside one joint turn (an equalizer over a shared id) — the construct legacy systems refused as too dangerous, now sound. crossCaveat_sound · driftStable_composes
